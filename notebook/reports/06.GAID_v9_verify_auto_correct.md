## 1. System Overview

The notebook fixes a recurring problem: rows in the training CSV reference image files that have since been **moved or reorganized** under `LOCAL_DATA_ROOT`. The pipeline:

1. **Verifies** every CSV path with `os.path.isfile`.
2. **Indexes** the on-disk dataset by basename to find where missing files actually live.
3. **Suggests** a per-folder mapping of `csv_folder → suggested_folder`.
4. **Auto-corrects** the CSV using either:
   - an accurate **per-row** lookup (Report 3), or
   - a fast **per-folder consensus** with fallbacks (Report 4).
5. **Visualizes** the resulting label / source-folder distribution.
6. **Audits** files on disk that are **not** referenced anywhere in the CSV.

End result: a corrected CSV with ~99.94% of `image_path` entries pointing to real files, plus distribution graphs and an "extras" report.

## 5. Pipeline Walkthrough

> Cell indices below match the notebook structure: cell 0 (loader), cell 1 (verify + suggest), cell 2 (per-row fix), cell 3 (folder-mapping fix), cell 4 (distribution), cell 5 (extras).

### 5.1 Cell 0 — Load source CSV

```python
import pandas as pd
df = pd.read_csv("…/GAID_Dataset_v9_full_Train.csv")
```

Loads the canonical training CSV into memory. All later cells either reuse this `df` (Reports 1–3) or re-load directly from disk (Reports 4–6) so they remain self-contained.

---

### 5.2 Report 1 + 2 — Verify & Suggest folder fixes

Cell 1 of the notebook does **two reports back-to-back**:

**Report 1 — Strict path check**

For every row in `df`:

```python
for img_path in df["image_path"]:
    if os.path.isfile(img_path):
        exist_image.append(img_path)
    else:
        missing_image.append(img_path)
```

Outputs a per-folder coverage HTML table (✓ fully present, ⚠ partially present, ✗ none present) with totals like:

```
Strict check (CSV path exists on disk):
  ✓ Found:   573,542
  ✗ Missing: 131,237
```

**Report 2 — Folder-level path suggestions**

1. Walks `LOCAL_DATA_ROOT` once and indexes every image by basename:

   ```python
   local_index[fname].append(Path(dirpath) / fname)
   ```

2. For every missing row, looks up its basename in the index → list of candidate folders.
3. Aggregates per CSV folder and picks the most common candidate as `suggested_folder` (rest are stored in `other_suggested_folders`).
4. Saves the result to `suggested_folders_<stem>.csv` and renders a colorized HTML table.

> **Run this cell first.** Reports 3 and 4 both depend on the suggestions CSV produced here.

---

### 5.3 Report 3 — Per-row auto-correct (accurate)

> **Recommended path-fix output for production use.**

Uses the in-memory `per_row` data from Report 2 (built from the basename index, so each suggestion is grounded in a real file on disk).

```python
correction_map[orig_path] = os.path.join(info["first_sugg_folder"], basename)
df_corrected["image_path"] = orig_paths.map(lambda p: correction_map.get(p, p))
```

- Rewrites only missing rows that have at least one candidate.
- Re-verifies every changed path.
- Saves to `<stem>_autocorrected.csv`.

Typical output:

```
rows changed                  : 130,835
changed → exists on disk      : 130,835
changed → still missing       :       0
TOTAL existing after fix      : 704,377
TOTAL missing  after fix      :     402   ← truly orphaned (no basename match anywhere)
coverage after fix            : 99.94 %
```

The 402 leftover rows correspond to images that genuinely no longer exist anywhere under `LOCAL_DATA_ROOT` (e.g. `Gen_samples_Gemini_100325/humangen_nature_070325`). They keep their original paths so you can decide what to do with them (drop, regenerate, etc.).

---

### 5.4 Report 4 — Folder-mapping auto-correct (full CSV, fast)

A **standalone** cell that does not require Reports 1–3 to be in memory — it only needs `SOURCE_CSV_PATH` and the previously-generated `suggested_folders_<stem>.csv`.

Algorithm:

1. Build a per-folder candidate list:
   ```
   folder_map[csv_folder] = [primary, fallback#1, fallback#2, …]
   ```
   where `primary` comes from `suggested_folder` and the rest are parsed out of `other_suggested_folders` (`/path(count); /path(count); …`).

2. For each row whose current `image_path` doesn't exist:
   - Try `primary/basename`, then each fallback in turn.
   - Use the first candidate that satisfies `os.path.isfile`.
   - If none exist, fall back to `primary/basename` as best-effort and flag the row.

3. Add a `path_status` column with values:

   | Status | Meaning |
   |---|---|
   | `ok-original` | Original path already existed → untouched |
   | `fixed-primary` | Replaced with primary suggestion (verified on disk) |
   | `fixed-fallback#k` | Replaced with the k-th alternate (verified on disk) |
   | `missing-after-rewrite` | Mapped but neither primary nor fallbacks contain the basename |
   | `missing-no-mapping` | Original folder not in `folder_map` (truly orphaned) |

4. Saves `<stem>_autocorrected_full.csv`.

This is the right cell to run when you want to **re-apply** the saved suggestions to a new copy of the source CSV without re-walking the filesystem.

> Set `OVERWRITE_SOURCE_CSV = True` to replace the source CSV in place.

### 5.5 Report 5 — Data distribution & graphs

Loads the chosen CSV (default `<stem>_autocorrected_full.csv`) and renders a 2×2 figure:

| Pane | Content |
|---|---|
| **(a)** Bar | Label distribution: `fake (AI)` vs `real (human)` with counts and % |
| **(b)** Pie | On-disk availability: `exists` vs `missing` |
| **(c)** Stacked bar | Top-N source folders (parent of `image_path`), stacked by label |
| **(d)** Stacked bar | High-level groups: `PubDB_Real`, `PubDB_Fake`, `Gen_samples`, `gen_ai_detector_dataset`, `real_faces_multinational_130325`, `ai_image_x_collection_feb25_cleaned`, `other` |

---

### 5.6 Report 6 — Extra files on disk not in the CSV

Audits the **inverse** of Report 1: files that exist on disk but are not referenced anywhere in the CSV.

```python
all_local  = set(every image file under LOCAL_DATA_ROOT)
csv_paths  = set(df["image_path"])
extras     = all_local - csv_paths
```

Outputs:

- Console: total extras and number of folders containing them.
- HTML: top-N folders by extra-file count (`extra_files`, `% of extras`).
- `extras_<stem>.csv`: every extra file as `path,folder`.
- `extras_per_folder_<stem>.csv`: aggregated `folder,extra_files,pct_of_extras`.

Use this to decide whether to:

- Add the extras to the CSV (and assign labels), or
- Delete them to reclaim disk space, or
- Keep them as a holdout pool.

## 6. Decision Flow — which output to use

```
                         ┌─ raw CSV (broken paths)
                         │
         Report 1 + 2 ───┼──> suggested_folders_<stem>.csv
                         │
              ┌──────────┴──────────┐
              │                     │
         Report 3              Report 4
       (per-row, slow)    (folder-map, fast)
              │                     │
              ▼                     ▼
   <stem>_autocorrected.csv   <stem>_autocorrected_full.csv
   (most accurate, 99.94%)    (also ~99.94% with fallbacks)
              │                     │
              └─────────┬───────────┘
                        ▼
                   Report 5  →  *_distribution.png
                   Report 6  →  extras_*.csv
```

| Use case | Choose |
|---|---|
| First run on a fresh CSV | Run 1+2 → 3, then 5 + 6 |
| Re-apply same suggestions to an updated CSV | Just Report 4 |
| Debugging / per-row tracing | Report 3 (it sees individual basename matches) |
| Production training data | `<stem>_autocorrected.csv` (Report 3) |

## 7. Common Operations

### Run the full pipeline from scratch

```text
1. Open notebook/06.GAID_v9_verify_auto_correct.ipynb
2. Execute cells top-to-bottom.
   - Cell 0   → load CSV
   - Cell 1   → strict check + folder suggestions   (~30s + filesystem walk)
   - Cell 2   → per-row auto-correct                 (fast, in-memory)
   - Cell 3   → folder-mapping auto-correct          (fast, file checks)
   - Cell 4   → distribution figure                  (~5s + optional disk check)
   - Cell 5   → extras audit                         (filesystem walk again)
```

### Apply pre-computed suggestions to a new CSV

1. Edit `SOURCE_CSV_PATH` in cell 3.
2. Make sure the matching `suggested_folders_<new_stem>.csv` exists next to it
   (or copy/rename an existing one).
3. Run only cell 3.

### Overwrite the source CSV in place

In cell 2 or 3, set:

```python
OVERWRITE_SOURCE_CSV = True
```

### Run the distribution / extras report on a different CSV

Edit `CSV_PATH` at the top of cell 4 and / or cell 5, then run that cell.

## 8. Troubleshooting & FAQ

**Q: REPORT 4's `_autocorrected_full.csv` had more missing rows (800) than REPORT 3's `_autocorrected.csv` (402). Why?**

REPORT 3 uses a **per-basename** index (one lookup per missing row, always points at a real file). REPORT 4 originally used a **per-folder consensus** which is right for most basenames in a folder but wrong for the long tail. The current REPORT 4 code parses `other_suggested_folders` and tries each candidate with `os.path.isfile`, bringing it back to ~99.94% coverage.

**Q: REPORT 1 reported 131,237 missing — REPORT 3 fixed 130,835. What about the 402?**

Those rows have basenames that don't exist anywhere under `LOCAL_DATA_ROOT`. Most are concentrated in `Gen_samples_Gemini_100325/humangen_nature_070325`. They retain their original (broken) paths in `_autocorrected.csv`.

**Q: `mv human/* .` says "Argument list too long".**

Use `find` instead, which avoids shell expansion:

```bash
find human/ -mindepth 1 -maxdepth 1 -exec mv -t . {} +
```

**Q: Cell 3 fails with `NameError: per_row not defined`.**

That cell is only valid right after cell 1 (Report 1 + 2) has been executed in the same kernel. For a kernel-independent run, use cell 3 (folder-mapping) instead.

**Q: I want to ignore a folder entirely during suggestion.**

In cell 1 you can drop rows from `missing_image` / `missing_labels` before the suggestion loop, e.g. by filtering on `csv_folder.startswith(".../skip_me")`.

## 9. Customization

### Add new image extensions

Both Report 2 (suggestions) and Report 6 (extras) use:

```python
_img_exts = {".jpg", ".jpeg", ".png", ".webp", ".bmp", ".tiff", ".tif"}
```

Add e.g. `".heic"` to that set if your dataset uses additional formats.

### Add a new top-level group to the distribution chart

Cell 4, edit the `top_group` mapping:

```python
df["top_group"] = df["image_path"].astype(str).map(
    lambda p: next((part for part in Path(p).parts
                    if part in {"PubDB_Real", "PubDB_Fake", "Gen_samples",
                                "gen_ai_detector_dataset",
                                "ai_image_x_collection_feb25_cleaned",
                                "real_faces_multinational_130325",
                                "MY_NEW_GROUP"}), "other")
)
```

### Speed up Report 6 by caching the disk index

If you also ran Report 1 in the same kernel, you can replace the `os.walk` block with:

```python
all_local = {str(p) for paths in local_index.values() for p in paths}
```

…instead of re-walking the filesystem.

## 10. Glossary

| Term | Definition |
|---|---|
| `LOCAL_DATA_ROOT` | Root directory under which all dataset images live (`data/processed/v9`) |
| `csv_folder` | The directory part of a row's `image_path` (one level above the file) |
| `suggested_folder` | The candidate directory the script believes the missing files were moved to |
| `basename` | The file's name without its directory (`image_05650.png`) |
| Strict check | `os.path.isfile(image_path)` — true only if the literal CSV path resolves on disk |
| Coverage % | `100 * found / total_rows` |
| Extras | Files on disk under `LOCAL_DATA_ROOT` that no row of the CSV references |
| `path_status` | Diagnostic column added by Report 4 explaining how each row was rewritten |
